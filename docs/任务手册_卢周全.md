## 🔍 任务二：混合检索模块（阶段一 ~ 阶段二初）

### 2.1 目标
在现有 FAISS 向量检索的基础上，增加 **BM25 稀疏检索** + **图检索**，实现三路混合检索。

### 2.2 背景
项目已有的检索方式：
- `src/vuln_analysis/tools/local_vdb.py` — FAISS 向量检索（稠密语义检索）

你需要新增：
- **BM25 检索** — 关键词精确匹配（用于精确搜索函数名、包名等）
- **图检索** — 基于知识图谱的关系检索（用于查找关联攻击链）

### 2.3 步骤

**第一步：BM25 稀疏检索**

推荐使用 `rank_bm25` 库：
```bash
uv add rank_bm25
```

创建文件 `src/vuln_analysis/tools/bm25_search.py`：
```python
from rank_bm25 import BM25Okapi

class BM25Searcher:
    """BM25 关键词检索。"""
    
    def __init__(self, documents: list[str]):
        tokenized = [doc.split() for doc in documents]
        self.bm25 = BM25Okapi(tokenized)
        self.documents = documents
    
    def search(self, query: str, top_k: int = 5) -> list[dict]:
        tokenized_query = query.split()
        scores = self.bm25.get_scores(tokenized_query)
        top_indices = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)[:top_k]
        return [{"doc": self.documents[i], "score": scores[i]} for i in top_indices]
```

**第二步：图检索**

基于任务一的知识图谱，创建 `src/vuln_analysis/tools/graph_search.py`：
```python
class GraphSearcher:
    """基于知识图谱的关系检索。"""
    
    def __init__(self, knowledge_graph):
        self.kg = knowledge_graph
    
    def search_related_vulns(self, cve_id: str) -> list[dict]:
        """通过 CVE → CWE → 找到同类漏洞。"""
        chain = self.kg.query_attack_chain(cve_id)
        related = []
        for cwe in chain.get("cwes", []):
            related.extend(self.kg.get_related_cves(cwe))
        return related
```

**第三步：混合检索融合**

创建 `src/vuln_analysis/tools/hybrid_search.py`，融合三路结果：
```python
class HybridSearcher:
    """三路混合检索: FAISS + BM25 + Graph。"""
    
    def __init__(self, faiss_searcher, bm25_searcher, graph_searcher):
        self.faiss = faiss_searcher
        self.bm25 = bm25_searcher
        self.graph = graph_searcher
    
    def search(self, query: str, cve_id: str = None, top_k: int = 5) -> list[dict]:
        """
        融合策略: 加权倒数排名融合 (Reciprocal Rank Fusion)
        """
        results = {}
        
        # 1. FAISS 语义检索
        for rank, item in enumerate(self.faiss.search(query, top_k)):
            key = item["doc"]
            results[key] = results.get(key, 0) + 1.0 / (rank + 60)
        
        # 2. BM25 关键词检索
        for rank, item in enumerate(self.bm25.search(query, top_k)):
            key = item["doc"]
            results[key] = results.get(key, 0) + 1.0 / (rank + 60)
        
        # 3. 图检索 (如果提供了 CVE ID)
        if cve_id and self.graph:
            for rank, item in enumerate(self.graph.search_related_vulns(cve_id)):
                key = str(item)
                results[key] = results.get(key, 0) + 1.0 / (rank + 60)
        
        # 按融合分数排序
        sorted_results = sorted(results.items(), key=lambda x: x[1], reverse=True)
        return [{"doc": doc, "score": score} for doc, score in sorted_results[:top_k]]
```

## 📝 任务三：安全报告生成模块（阶段二，2.23 - 3.8）

### 3.1 目标
开发报告自动生成模块，支持 **Markdown + JSON + OpenVEX** 三种输出格式。

### 3.2 背景
项目已有的输出逻辑在 `src/vuln_analysis/functions/cve_file_output.py`，但格式单一。
你需要将其增强为支持多格式的报告生成器。

### 3.3 步骤

创建 `src/vuln_analysis/report/` 目录：
```
src/vuln_analysis/report/
├── __init__.py
├── generator.py        # 报告生成器主入口
├── markdown_report.py  # Markdown 格式
├── json_report.py      # JSON 格式
└── openvex_report.py   # OpenVEX 格式
```

**Markdown 报告**应包含：
- 扫描摘要（镜像名、CVE 总数、风险分布）
- 每个 CVE 的详细分析（情报、Agent 判定、依据）
- 修复建议

**OpenVEX 报告**需遵循 [OpenVEX 规范](https://github.com/openvex/spec)：
```json
{
  "@context": "https://openvex.dev/ns/v0.2.0",
  "@id": "https://containerguard.ai/vex/...",
  "statements": [
    {
      "vulnerability": {"name": "CVE-2023-36632"},
      "products": [{"@id": "pkg:docker/morpheus@23.11"}],
      "status": "not_affected",
      "justification": "vulnerable_code_not_present"
    }
  ]
}
```

### 3.4 交付物
- `src/vuln_analysis/report/` 目录下的完整代码
- 至少生成一份样例报告（三种格式各一份）
